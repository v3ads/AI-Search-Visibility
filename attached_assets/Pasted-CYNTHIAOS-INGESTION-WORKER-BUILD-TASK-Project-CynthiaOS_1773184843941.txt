CYNTHIAOS INGESTION WORKER BUILD TASK

Project: CynthiaOS
Database: Neon Postgres
Project: cynthiaos-spine
Branch: staging

Goal:
Build the minimal ingestion worker required to accept webhook events and insert them into the bronze_events table.

Architecture requirements:

The worker must perform the following sequence for every request:

1. Receive HTTP POST request
2. Authenticate request using shared secret header
3. Validate ingestion envelope
4. Compute payload hash
5. Insert event into bronze_events
6. Return structured response

Technology choice:

Use:
Node.js + TypeScript
Lightweight framework (Hono or Express)
pg driver for Postgres

Worker structure:

src/
index.ts
auth.ts
validate.ts
hash.ts
db.ts
types.ts

Functional requirements:

Endpoint:
POST /ingest

Authentication:
Header: X-Cynthia-Signature

Use HMAC-SHA256 signature over the raw request body using environment variable:

INGEST_SECRET

Envelope format:

{
  "source": "appfolio",
  "source_entity": "tenant_directory",
  "event_type": "report_snapshot",
  "event_version": "1.0",
  "occurred_at": "timestamp",
  "idempotency_key": "string",
  "payload": {},
  "metadata": {}
}

Database connection:

Use environment variable:

DATABASE_URL

Insert logic:

INSERT INTO bronze_events (
  source,
  source_entity,
  event_type,
  event_version,
  occurred_at,
  idempotency_key,
  payload,
  payload_hash,
  size_bytes,
  metadata
)

Return response:

{
  "status": "accepted",
  "event_id": "<uuid>"
}

Project output required:

1. Full project folder structure
2. All TypeScript source files
3. package.json
4. instructions to run locally
5. instructions to deploy to staging

Important constraints:

- Do NOT modify database schema
- Worker must be stateless
- Worker must only write to bronze_events
- No background jobs or schedulers
- No business logic beyond ingestion

Return the complete worker codebase.
